All About Javascript functions

Remember: Javascript Functions are objects

Unlike other languages, JS functionsare objects. This means they are like any other value.

  • They can be stored in variables, ojects and arrays
  • Passed as arguments to other functions, these are called as callbacks aka high order functions
  • can be returned from other functions

There are two ways of creating a function: function declaration and function expression. There are some key differences betweeen the two, I will write about this more when I talk about Hoisting Below is a simple function declaration example:
function add(num1, num2) { var x= num1 +num2; return x; } var result=add(2,3); console.log(result);

Lets create a object
var myobjects = { name:"This objects has many properties aka methods and their associated values", num_of_properties: 3, this_prop_isaFunc: function() {alert ("I belong to th object called myobjects"); } }; myobjects.this_prop_isaFunc();

Two examples of a function expression below
var mult = function awesome(a,b) { return (a*b); }; var mult2 = function (a,b) { return (a*b); }; As you can see function expression differs from function declaration, here we are passing a function to a variable, which stores the entire function, this is makes a difference during call time---more about it when I write about hoisting. Also the two examples are different in the way we have defined the function. The second example is an anyomyomus function(notice no name after the keyword function). The word that comes after function is optional, you can skip it, depending on preference

you can invoke both methods by doing the following: console.log('Running mult(3,5)', mult(3,5)); console.log('Running mult2(10,3)', mult2(10,3));

what is the difference between these two objects? console.dir(mult); console.dir(mult2); check the console and see where javascript stored the named of the function. The main difference is that if you want to see MORE detail of an object then console.log is showing you, try console.dir.
